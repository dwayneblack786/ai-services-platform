RELATIONSHIPS:
--------------
1. customers (1) ──► (1) assistant_channels
   - One customer has one channel configuration

2. assistant_channels (1) ──► (*) assistant_calls
   - One channel configuration generates many voice calls

3. assistant_channels (1) ──► (*) assistant_chats
   - One channel configuration generates many chat sessions

4. customers (1) ──► (*) subscriptions
   - One customer can have multiple product subscriptions

5. products (1) ◄── (*) subscriptions
   - One product can be subscribed to by many customers

6. subscriptions (1) ──► (*) invoices
   - One subscription generates many invoices

7. customers (1) ──► (*) invoices
   - One customer receives many invoices

8. subscriptions (1) ──► (*) usage_events
   - One subscription tracks many usage events

9. customers (1) ──► (*) users
   - One customer can have multiple user accounts

10. customers (1) ──► (*) api_keys
    - One customer can have multiple API keys

KEY INDEXES:
------------
Performance-critical indexes:
✓ assistant_channels: customerId (unique), voice.phoneNumber (unique)
✓ assistant_calls: (customerId, startTime) compound index for call logs
✓ assistant_chats: (customerId, startTime) compound index for chat history
✓ invoices: (customerId, createdAt) compound index for billing history
✓ usage_events: (customerId, timestamp) compound index for usage tracking
✓ users: email (unique)
✓ api_keys: key (unique)
