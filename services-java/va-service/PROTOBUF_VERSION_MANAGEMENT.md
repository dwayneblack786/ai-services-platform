# Protobuf Version Management

## Current Configuration
- **Protobuf Version**: 4.29.1
- **gRPC Version**: 1.75.0
- **Java Version**: 21

## Version Compatibility Matrix

| Protobuf | gRPC    | Java | Status |
|----------|---------|------|--------|
| 4.29.1   | 1.75.0  | 21   | ✅ Current |
| 4.28.3   | 1.75.0  | 21   | ❌ Missing `isStringEmpty()` |
| 4.29.3   | 1.75.0  | 21   | ⚠️  May have `blockingV2UnaryCall()` issues |

## Why Version Alignment Matters

### The Three Components
1. **protoc** (compiler): Generates Java code from `.proto` files
2. **protobuf-java** (runtime): Provides base classes like `GeneratedMessage`
3. **grpc-java**: RPC framework that depends on both

### Common Issues
- **`isStringEmpty() undefined`**: protoc 4.28.3+ generates calls to this method, but runtime 4.28.3 doesn't have it. Fixed in 4.29.1+.
- **`blockingV2UnaryCall() undefined`**: Newer protoc generates calls that older gRPC versions don't support.

## How to Ensure Consistency

### 1. Use Maven Enforcer (Added to pom.xml)
```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-enforcer-plugin</artifactId>
    <configuration>
        <rules>
            <requireProperty>
                <property>protobuf.version</property>
                <regex>4\.29\.[0-9]+</regex>
            </requireProperty>
        </rules>
    </configuration>
</plugin>
```

### 2. Run Verification Script
```powershell
# After every build
.\verify-protobuf.ps1
```

### 3. Clean Maven Cache on Version Changes
```powershell
# When changing versions, clear cache
Remove-Item -Recurse -Force "$env:USERPROFILE\.m2\repository\com\google\protobuf\protoc"
.\mvnw.cmd clean compile
```

### 4. Check Generated Code
The first lines of generated files should show:
```java
// Generated by the protocol buffer compiler.  DO NOT EDIT!
// NO CHECKED-IN PROTOBUF GENCODE
// source: voice.proto
// Protobuf Java Version: 4.29.1
```

## Build Process

### Dependency Management Strategy

The project uses Maven's `<dependencyManagement>` section to enforce consistent versions:

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.google.protobuf</groupId>
            <artifactId>protobuf-java</artifactId>
            <version>${protobuf.version}</version>
        </dependency>
        <dependency>
            <groupId>com.google.errorprone</groupId>
            <artifactId>error_prone_annotations</artifactId>
            <version>2.41.0</version>
        </dependency>
    </dependencies>
</dependencyManagement>
```

This prevents gRPC from pulling incompatible Protobuf versions (e.g., gRPC 1.75.0 → protobuf-java 3.25.8).

### Maven Enforcer Validation

The build automatically validates constraints using maven-enforcer-plugin:

1. **RequireJavaVersion**: Java 21+
2. **DependencyConvergence**: All transitive dependencies use consistent versions
3. **RequireProperty (protobuf.version)**: Matches `4\.29\.[0-9]+`
4. **RequireProperty (grpc.version)**: Matches `1\.75\.[0-9]+`

Run validation: `.\mvnw.cmd validate` (all rules must pass)

### Clean Build (Recommended after version changes)
```bash
# 1. Update pom.xml versions
# 2. Validate enforcer rules
.\mvnw.cmd validate

# 3. Clear Maven cache for protoc
Remove-Item -Recurse -Force "$env:USERPROFILE\.m2\repository\com\google\protobuf\protoc"

# 4. Clean and rebuild
.\mvnw.cmd clean compile

# 5. Verify
.\verify-protobuf.ps1
```

### Normal Build
```bash
.\mvnw.cmd compile
```

## Troubleshooting

### Problem: "Method isStringEmpty() is undefined"
**Cause**: protoc version > protobuf-java runtime version
**Solution**: Upgrade `<protobuf.version>` in pom.xml to 4.29.1+

### Problem: "Method blockingV2UnaryCall() is undefined"  
**Cause**: protoc generates code for newer gRPC than installed
**Solution**: Align gRPC and Protobuf versions per compatibility matrix

### Problem: Old protoc cached in Maven
**Cause**: Maven downloads protoc once and caches it
**Solution**: 
```powershell
Remove-Item -Recurse -Force "$env:USERPROFILE\.m2\repository\com\google\protobuf\protoc\<old-version>"
```
